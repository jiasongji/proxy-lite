Proxy Lite 部署运维手册

轻量 A/B 中转落地脚本：服务器 A 只提供 AnyTLS / NaiveProxy 用户入口；服务器 B 只提供 Shadowsocks landing。AnyTLS 与 NaiveProxy 可分别选择 direct 或 egress-b 出口。

1. 概览 2. 环境要求 3. 安装 4. A/B 拓扑 5. 命令 6. 升级回退 7. 审计 8. 安全

1. 概览

Proxy Lite 是完整 proxy-manager 的独立轻量项目，去掉多用户、流量限制和域名/IP 分流功能，只保留 A/B 中转落地与协议级出站选择。

项目值
短命令PL（大写）
完整命令proxy-lite
GitHubhttps://github.com/jiasongji/proxy-lite
Latest Releasev0.1.1
默认镜像ghcr.io/sagernet/sing-box:latest

保留 AnyTLS、NaiveProxy、A-B 内部 Shadowsocks、Docker host network、真实 sing-box check -c、安全升级和失败回退。

裁剪 user / route / stats / traffic / quota；不创建 config/users.json；不生成 route.rule_set。

2. 环境要求

  • root 用户执行生产运维命令。
  • 已安装 Docker 与 Docker Compose。
  • 服务器 A 如启用 AnyTLS / NaiveProxy，需要可用 TLS 证书与私钥路径。
  • 服务器 B 的 Shadowsocks 端口建议仅允许服务器 A 的公网 IP 访问。

3. 安装

快速安装

curl -fsSL https://github.com/jiasongji/proxy-lite/releases/latest/download/proxy-lite.sh -o /tmp/proxy-lite.sh
bash /tmp/proxy-lite.sh install

服务器 B：Shadowsocks landing

PL install --yes \
  --node-role egress_b \
  --domain b.example.com \
  --server-ip 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

服务器 A：只部署 AnyTLS direct

PL install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components anytls \
  --anytls-egress direct \
  --cert-file /path/fullchain.pem \
  --key-file /path/privkey.pem \
  --anytls-port 30001

服务器 A：只部署 NaiveProxy 经 B

PL install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components naive \
  --naive-egress egress-b \
  --cert-file /path/fullchain.pem \
  --key-file /path/privkey.pem \
  --naive-port 30002 \
  --b-ss-host 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

服务器 A：AnyTLS direct + NaiveProxy 经 B

PL install --yes \
  --node-role entry_a \
  --domain a.example.com \
  --server-ip 203.0.113.10 \
  --components anytls,naive \
  --anytls-egress direct \
  --naive-egress egress-b \
  --cert-file /path/fullchain.pem \
  --key-file /path/privkey.pem \
  --anytls-port 30001 \
  --naive-port 30002 \
  --b-ss-host 198.51.100.20 \
  --b-ss-port 30003 \
  --b-ss-method aes-128-gcm \
  --b-ss-password '<B_SS_PASSWORD>'

4. A/B 拓扑

entry_a 上的用户入口只可能是 anytls-in 与 naive-in。Shadowsocks 只出现在 A 的 egress-b outbound 与 B 的 ss-landing-in inbound。

组合参数效果
AnyTLS direct--components anytls --anytls-egress direct用户连 A，出口为 A。
NaiveProxy 经 B--components naive --naive-egress egress-b用户连 A，出口为 B。
混合--anytls-egress direct --naive-egress egress-bAnyTLS 从 A 出口，NaiveProxy 从 B 出口。
全部经 B--anytls-egress egress-b --naive-egress egress-b两种入口都经 B。
全部 direct--anytls-egress direct --naive-egress direct两种入口都从 A 出口，不要求 B 参数。
{
  "route": {
    "rules": [
      { "inbound": ["anytls-in"], "action": "route", "outbound": "direct" },
      { "inbound": ["naive-in"], "action": "route", "outbound": "egress-b" }
    ],
    "final": "direct"
  }
}
只有至少一个协议选择 egress-b 时，服务器 A 才需要 --b-ss-host、--b-ss-port、--b-ss-method、--b-ss-password。

7. 审计

  • bash -n proxy-lite.sh 与 ShellCheck。
  • CLI help / menu smoke。
  • 生成配置矩阵：AnyTLS only、NaiveProxy only、混合出站、全部 direct、全部经 B、B landing。
  • 对抗测试：拒绝 A 侧 --components ss，旧 ENABLE_SS=1 不会生成用户 SS 入站，经 B 缺少 B 参数时失败。
  • Docker sing-box check -c 真实检查。
  • 升级/回退 smoke：有效镜像通过，无效镜像失败并恢复快照。

8. 安全

  • B_SS_PASSWORD 是 A-B 内部链路密码，不提供给终端用户。
  • 客户端目录只导出 AnyTLS / NaiveProxy 配置和 full test client。
  • 公开文档仅使用占位 IP、域名、端口和密码。
  • B landing 端口建议在云安全组、防火墙或面板中限制为仅允许 A 的公网 IP。
